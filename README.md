# PortMap — карта портов свитчей

Веб-приложение для GitHub Pages: список свитчей → карта портов (VLAN, IP, MAC, хост, статус, камеры) → QR-коды.
Сканируешь QR на свитче телефоном — открывается **просмотр** карты именно этого свитча. Редактирование — только для админа.

## Файлы

| Файл            | Что это                                          |
|-----------------|--------------------------------------------------|
| `index.html`    | Точка входа                                       |
| `data.js`       | Данные: свитчи + справочник VLAN                  |
| `app.js`        | Логика                                            |
| `styles.css`    | Оформление (светлая/тёмная тема)                  |
| `qrcode.min.js` | Генератор QR                                      |

## Выложить на GitHub Pages

1. Создай репозиторий, залей все файлы.
2. **Settings → Pages → Source:** ветка `main`, папка `/ (root)` → Save.
3. Через ~1 минуту сайт по адресу `https://ТВОЙ-ЛОГИН.github.io/РЕПО/`.

## Что умеет

**Тема** — кнопка ☀/🌙 вверху. Светлая и тёмная, выбор запоминается.

**Шрифт** — шестерёнка → Настройки → Шрифт. Несколько готовых шрифтов (с кириллицей) или **загрузи свой** (.woff2/.ttf/.otf).

**Камеры** — порты-камеры (VLAN с названием CAMERA или с каналом D4/D19…) подсвечены голубым и со значком камеры. Индикаторы активных портов мигают.

**VLAN-подсказки** — нажми на бейдж «VLAN 3» — выскочит название (например CAMERA). Справочник VLAN правится в админке.

## Гость и Админ

- **Гость** (любой, кто сканирует QR) видит только просмотр. Никаких кнопок редактирования.
- **Админ** включается так: шестерёнка → Настройки → Администрирование → задай PIN → «Создать PIN и войти». PIN запоминается на твоём устройстве. После этого появляются кнопки добавить/изменить/удалить свитчи, порты и VLAN.

> PIN — это лёгкая защита от случайных нажатий (сайт публичный, данные в нём всё равно открыты). Настоящий ключ к записи на GitHub — токен (ниже).

## Изменения сразу на GitHub

Чтобы правки из приложения попадали прямо в репозиторий (и их видели все):

1. Создай **Fine-grained Personal Access Token**: GitHub → Settings → Developer settings → Fine-grained tokens →
   доступ только к твоему репозиторию, право **Contents: Read and write**.
2. В приложении: шестерёнка → Настройки → Сохранение на GitHub → заполни owner, repo, ветку, путь (`data.js`) и вставь токен → «Сохранить» (можно «Проверить связь»).
3. Теперь в админ-панели есть кнопка **«На GitHub»** — жмёшь, и `data.js` коммитится в репозиторий. Сайт обновится через ~1 минуту.

Токен хранится только в твоём браузере (localStorage), в код и в репозиторий не попадает.

Без токена можно работать по-старому: правишь в приложении → **«Скачать data.js»** → заливаешь файл в репозиторий вручную. Кнопка **«Сбросить»** отменяет локальные правки.

## Добавить свитчи вручную (через data.js)

В `data.js` скопируй блок свитча и поменяй данные. Поля порта: `port`, `vlan`, `ip`, `mac`, `host`,
`channel` (канал камеры, напр. `D4`), `off` (`true` — порт выключен, красный).
Справочник VLAN — объект `window.VLANS` (номер: "название").
